# 사내 복지몰 스타일가이드

## 한 줄 방향
사내 복지몰은 "따뜻한 복지 경험"과 "차분한 운영 신뢰감"을 동시에 주는 디자인 시스템으로 정리한다.

## 현재 문제 진단
- 사용자 화면은 웜톤, 글래스 효과, 강한 장식 요소가 많고 관리자 화면은 차갑고 평면적이라 같은 서비스처럼 보이지 않는다.
- 분홍, 주황, 파랑, 네이비가 동시에 강하게 쓰여 중심 색이 흐려진다.
- 화면마다 카드 모양, 그림자, 버튼 높이, 코너 반경이 제각각이다.
- 복지몰 특성상 "잔액", "포인트", "주문 상태", "관리 액션"이 중요하지만 시각적 위계가 일정하지 않다.

## 브랜드 원칙

### 1. 톤 앤 매너
- 사용자 영역은 따뜻하고 친절해야 한다.
- 관리자 영역은 단정하고 신뢰감 있어야 한다.
- 두 영역 모두 같은 브랜드 컬러와 같은 타이포 스케일을 쓴다.
- 장식보다 정보 전달이 우선이다.

### 2. 키워드
- Warm
- Clear
- Trustworthy
- Practical

### 3. 금지 원칙
- 과한 유리 효과 사용 금지
- 의미 없는 강한 그라디언트 남용 금지
- 페이지마다 다른 라운드 값 사용 금지
- 상태색을 브랜드색처럼 혼용 금지

## 디자인 시스템 구조

### 사용자 영역
- 목적: 상품 탐색, 포인트 확인, 주문 추적
- 인상: 따뜻함, 가벼움, 생활형 커머스 감성
- 원칙: 정보보다 분위기가 앞서지 않게 한다

### 관리자 영역
- 목적: 운영, 조회, 입력, 상태 변경
- 인상: 차분함, 밀도감, 정확성
- 원칙: 장식보다 처리 효율이 앞서야 한다

### 사용 규칙

- 상태 표현은 성공/경고/위험/정보 색으로만 처리한다.
- 사용자 화면 배경은 `Surface / App Background` 를 기본으로 한다.
- 관리자 화면 배경은 `Surface / Admin Background` 를 기본으로 한다.
- 진한 네이비는 본문 배경으로 쓰지 않고 제목, 차트, 포인트 강조 보조색으로만 사용한다.

## 타이포그래피

### 폰트
- Web: `SUIT`, `Pretendard`, `system-ui`, `sans-serif`
- React Native: 플랫폼 시스템 폰트 사용, 대신 동일한 크기/두께 체계를 유지

### 타입 스케일
- `Display 28 / 700`
- `Title 24 / 700`
- `Heading 20 / 700`
- `Section 18 / 700`
- `Body 16 / 500`
- `Body Small 14 / 500`
- `Caption 12 / 600`
- `Micro 11 / 700`

### 사용 규칙
- 한 화면에서 제목 계층은 최대 3단계만 사용한다.
- 본문 기본값은 `16 / 500`, 관리자 테이블/폼은 `14 / 500` 를 사용한다.
- 숫자 정보는 가능한 한 semibold 이상으로 처리한다.
- 포인트, 금액, 수량은 숫자 가독성을 위해 문자 간격을 좁히지 않는다.

## 간격 시스템

### 기본 단위
- 4px 기반 스케일 사용
- `4, 8, 12, 16, 20, 24, 32, 40, 48, 64`

### 레이아웃 권장값
- 모바일 좌우 여백: `16`
- 모바일 섹션 간격: `24`
- 카드 내부 패딩: `16` 또는 `20`
- 관리자 페이지 패딩: `24`
- 테이블 상하 여백: `16`

## 라운드 / 테두리 / 그림자

### Radius
- `XS`: `8`
- `SM`: `12`
- `MD`: `16`
- `LG`: `20`
- `XL`: `28`

### Border
- 기본 보더: `1px solid #DEE3EA`
- 강조 보더: `1px solid #F4C9C3`

### Shadow
- `Card / Soft`: `0 8px 24px rgba(31, 36, 48, 0.06)`
- `Card / Hover`: `0 14px 32px rgba(31, 36, 48, 0.10)`
- `Modal`: `0 24px 48px rgba(31, 36, 48, 0.16)`

### 규칙
- 글래스 블러 효과는 사용하지 않는다.
- 기본 카드는 흰색 배경 + 얕은 그림자 + 얇은 보더 조합으로 통일한다.
- 카드마다 서로 다른 radius를 쓰지 않는다.

## 아이콘 / 일러스트 / 이미지

### 아이콘
- 선명한 라인 아이콘 또는 단색 아이콘 사용
- 기본 두께는 1.5px 또는 2px로 통일
- 상태 아이콘은 상태 색만 사용

### 상품 이미지
- 썸네일 비율은 `1:1` 또는 `4:5` 만 허용
- 배경이 복잡한 이미지는 카드 배경과 충돌하지 않게 여백 프레임을 둔다

### 일러스트
- 과한 3D 일러스트보다 단순한 라인/플랫 스타일 사용
- 빈 상태 화면에서만 보조적으로 사용

## 컴포넌트 가이드

### 1. 버튼

#### Primary
- 높이: `48`
- 배경: `Brand / Primary`
- 텍스트: `Neutral / 0`
- radius: `16`

#### Secondary
- 높이: `48`
- 배경: `Neutral / 0`
- 보더: `Neutral / 300`
- 텍스트: `Neutral / 800`

#### Tertiary
- 높이: `40`
- 배경: `Brand / Primary Soft`
- 텍스트: `Brand / Primary`

#### 금지
- 한 화면에 서로 다른 높이의 메인 CTA 2개 사용 금지
- 그림자 강한 버튼 금지

### 2. 인풋 / 셀렉트 / 검색
- 높이: `48`
- 배경: `Neutral / 0`
- 보더: `Neutral / 300`
- 포커스: `Brand / Primary` 1px 강조
- 에러: `Danger` 보더 + 메시지 노출
- 도움말 텍스트는 `Body Small`

### 3. 카드

#### 일반 카드
- 배경: `Surface / Page`
- radius: `20`
- padding: `16` 또는 `20`

#### 잔액 카드
- 사용자 영역에서 가장 강한 정보 카드
- 배경은 브랜드 단색 또는 아주 약한 웜 그라디언트만 허용
- 금액과 잔액 정보가 장식보다 우선

#### 관리자 요약 카드
- 평면형 카드 사용
- 아이콘보다 숫자와 라벨 우선

### 4. 칩 / 배지 / 상태 라벨
- 높이: `28`
- 좌우 패딩: `10-12`
- radius: `999`
- 상태 배지는 상태 컬러 soft 배경 + 진한 텍스트 조합

예시
- `주문 완료`: Info Soft + Info
- `배송 중`: Warning Soft + Warning
- `지급 완료`: Success Soft + Success
- `취소/반품`: Danger Soft + Danger

### 5. 탭
- 텍스트 중심 탭 사용
- 하단 라인 또는 배경 강조 중 하나만 쓴다
- 사용자 영역은 라인형, 관리자 영역은 세그먼트형 사용 가능

### 6. 테이블
- 관리자 전용
- 행 높이: `52-56`
- 헤더 배경: `Neutral / 100`
- 헤더 텍스트: `Body Small / 600`
- 행 hover는 아주 연한 회색만 사용
- 액션 버튼은 우측 정렬

### 7. 모달 / 바텀시트
- 모바일은 바텀시트 우선
- 관리자 확인 액션은 모달 우선
- 파괴적 액션에는 보조 설명 문구 필수

## 화면 패턴 가이드

### 사용자 홈
- 상단에 브랜드 히어로를 두되 높이는 과하지 않게 제한한다
- 가장 먼저 보여야 할 정보는 `현재 포인트`, `추천 상품`, `주문 상태`
- 이벤트 배너보다 잔액과 진입 CTA를 우선 배치한다

### 상품 목록
- 검색바와 필터는 상단 고정
- 카드형 리스트 기준으로 운영
- 정보 우선순위는 `이미지 > 상품명 > 포인트 가격 > 상태`

### 상품 상세
- 상품 정보와 구매 액션을 분리한다
- 가격, 재고, 배송 정보는 접지 말고 바로 보여준다

### 주문 / 배송
- 주문 상태 타임라인은 단순하게 유지한다
- 상태 텍스트와 배지는 항상 같은 규칙으로 사용한다

### 관리자 대시보드
- KPI 카드, 작업 대기, 최근 주문 순으로 배치
- 차트는 있어도 1~2개만 사용
- 운영 액션은 목록 화면에서 빠르게 연결돼야 한다

### 관리자 폼 화면
- 2열 이상 그리드는 데스크톱에서만 사용
- 필수값, 도움말, 검증 메시지를 같은 위치에 둔다

## 모션 가이드
- 전환 시간: `160ms - 220ms`
- 이징: `ease-out`
- 화면 진입은 짧은 페이드 + 8px 정도의 이동만 허용
- 장바구니 추가, 상태 변경, 저장 완료 같은 순간 피드백만 애니메이션 사용
- 불필요한 패럴랙스, 과한 스케일 애니메이션 금지

## 접근성 가이드
- 본문 텍스트 대비는 최소 `4.5:1`
- 상태를 색만으로 구분하지 않는다
- 버튼과 탭의 터치 영역은 최소 `44x44`
- 입력 에러는 색 + 문구로 함께 안내한다
- 관리자 화면에서 표와 액션 버튼의 키보드 포커스가 분명해야 한다

## React / React Native 적용 원칙

### 공통
- 색상, 간격, radius, 타이포를 토큰화한다
- 화면마다 직접 값 하드코딩하지 않는다
- 상태색과 브랜드색을 분리한다

### React 웹
- CSS 변수 또는 theme 객체로 토큰 관리
- 관리자와 사용자 영역은 같은 토큰을 공유하되 surface만 분리

### React Native
- 그림자는 플랫폼 차이를 고려해 최소화
- blur, glass 효과를 기본 시스템에서 제외
- 웹과 같은 구조의 spacing, font size, color name을 사용

## 권장 토큰 예시

```css
:root {
  --color-brand-primary: #f05a4a;
  --color-brand-primary-soft: #fff1ee;
  --color-text-primary: #1f2430;
  --color-text-secondary: #515b6b;
  --color-border-default: #dee3ea;
  --color-surface-page: #ffffff;
  --color-surface-app: #fbf7f2;
  --color-surface-admin: #f7f9fc;
  --color-success: #1f9d61;
  --color-warning: #d58a1f;
  --color-danger: #d94a5a;
  --radius-sm: 12px;
  --radius-md: 16px;
  --radius-lg: 20px;
  --space-1: 4px;
  --space-2: 8px;
  --space-3: 12px;
  --space-4: 16px;
  --space-5: 20px;
  --space-6: 24px;
}
```

## 빠른 적용 우선순위
1. 컬러 토큰부터 통일한다.
2. 버튼 높이와 카드 radius를 통일한다.
3. 사용자/관리자 배경과 카드 스타일을 분리하되 브랜드색은 유지한다.
4. 상태 배지 규칙을 먼저 고정한다.
5. 잔액 카드, 상품 카드, 관리자 테이블 3가지를 핵심 컴포넌트로 먼저 시스템화한다.

## 최종 디자인 판단 기준
- 이 화면이 "복지몰"처럼 따뜻한가
- 동시에 "사내 서비스"처럼 신뢰감 있는가
- 브랜드색 없이도 정보 위계가 읽히는가
- 사용자 화면과 관리자 화면이 같은 서비스로 보이는가
